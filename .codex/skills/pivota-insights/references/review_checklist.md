# Pivota Insights Review Checklist

Use this checklist when marking each row `pass` or `rewrite`.

## Pass Standard

- `What it is` says what the product is in one tight sentence.
- `What it is` names the product type and the core functional angle.
- `Why it stands out` explains a real product difference, not just the packaging, size, or application format.
- The copy sounds normalized and editorial, not like seller copy.
- Seller-only rows stay conservative and grounded in product facts.

## Rewrite Triggers

Mark `rewrite` if any of these appear:

- Generic merchandising:
  - `jumbo size`
  - `double up and save`
  - `value pack`
- Abstract positioning:
  - `positioning`
  - `story`
  - `role`
  - `format`
- Low-signal template lines:
  - `daily moisture plus brightening support`
  - `glow and hydration in one serum`
  - `targeted under-eye application`
  - `dedicated dark-spot serum`
- Repetition:
  - highlight repeats `What it is` without adding a new reason
  - highlight repeats legacy `Overview`
- Unsupported claims:
  - community sentiment on seller-only rows
  - overly strong efficacy language without clear grounding

## Field-by-Field Standard

### What it is

- Should usually be 1 sentence.
- Should be concrete, not promotional.
- Should front-load `product type + key actives/approach + main concern`.

### Why it stands out

- Prefer 1 to 2 bullets.
- Each bullet must say why this item is meaningfully different from a plain baseline in the same step.
- Good differences:
  - active pairing
  - treatment scope
  - step compression
  - wear profile when materially relevant
- Weak differences:
  - obvious placement
  - vague “support”
  - generic texture words with no real implication

### Seller-only rows

- No `people love`, `users say`, or implied community evidence.
- No invented comparison to products not in evidence.
- If the only distinctive thing is size, packaging, or convenience, the row is not ready.
